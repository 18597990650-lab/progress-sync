## 1. Demo 简介

**进度同步器** 是一个面向职场团队的 Web 应用（单页 HTML），核心解决"员工不敢同步进度、上级看不到项目进展"的沟通困境。

**面向谁：** 刚入职场 1-3 年的新人、性格内向不善于向上沟通的团队成员（i 人为主），以及需要管理 5-15 人小团队的负责人。

**核心功能：**

- **员工端 — 进度同步"安全区"**：每个任务用进度条 + 阶段标签可视化呈现，点击「更新进度」弹出表单，通过滑条调百分比、点选当前阶段、选用快捷话术模板（"进展顺利""遇到难点""需要支持"），30 秒完成一次同步。首屏横幅直接传达"同步进度，是一件很勇敢的事"，从心理层面消解"怕打扰"的障碍。
- **上级端 — 团队进度看板**：一屏展示团队平均进度、本周同步次数、需关注人数三个核心指标；柱状图直观对比各成员进度；三个标签页（全部任务 / 按成员查看 / 需关注）灵活切换视角；发现滞后任务可一键"鼓励"而非质问。
- **更新记录时间线**：每个任务支持展开查看完整的进度变更历史（时间、阶段、百分比、文字说明），员工可逐条删除误操作记录；每次提交新进度自动追加时间线条目，历史变更一目了然。

## 2. Demo 创作思路

**灵感来源：** 观察到身边的内向型同事有一个反复出现的动作——打开和上级的聊天窗口，打了几行字又全部删掉，最后关闭窗口什么都没发。他们不是不想沟通，而是"怕打扰"这三个字堵住了出口。

**想解决的问题：** 传统教育体系注重个人能力培养，却几乎不教"如何向上沟通"。结果是大量职场新人陷入"员工不敢问、领导不知道"的双向信息盲区：员工因为心理负担放弃同步，上级因为信息缺失只能逐个追问，而追问又进一步加重了员工的压力，形成恶性循环。

**为什么做这个方向：** 市面上的项目管理工具（飞书、钉钉、Jira）都是"自上而下"的管理视角，强调任务分配和截止日期追踪，对"执行者主动同步"这个动作没有任何心理层面的设计支持。我希望做一个"自下而上"的工具，核心不是管理而是"鼓励表达"——用温暖的设计语言、正向的反馈机制、极低的操作门槛，让同步进度变成一件自然且被鼓励的事，而不是一次心理博弈。

## 3. Demo 体验地址

本 Demo 为交互式 HTML 文件，下载附件中的 **index.html** 后直接用浏览器打开即可体验。

**体验指引：**
- 顶部导航栏可切换「我是员工」和「我是上级」两个视角
- 员工视角：点击任务卡片的「更新进度」按钮体验提交流程；点击「更新记录」展开时间线，悬停记录可见删除按钮
- 上级视角：点击三个标签页切换不同查看维度；点击任务右侧「鼓励一下」体验正向反馈
