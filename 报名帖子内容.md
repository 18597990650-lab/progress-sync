**【标签】** 学习工作

**【标题】** 学习工作赛道 · 进度同步器 —— 为i人打造的职场进度同步"安全区"

**【正文】**

## 1. 创意名称 + 创意介绍

**创意名称：进度同步器**

- **想解决什么问题：** 在传统教育模式下成长的职场新人普遍缺乏向上沟通的经验，尤其是内向型（i人）员工，虽然内心想同步项目进度，却因"怕打扰上级"而不敢开口；与此同时，上级也因为没有及时反馈，无法准确掌握项目推进状态，只能通过反复追问来了解情况，导致双方都陷入沟通焦虑。

- **为什么会想到做这个：** 身边的许多同事（尤其是性格内向的人）都有一个共同的困扰：完成一项阶段性工作后，想告诉上级进展，但反复打开聊天窗口又关掉——"发这条消息会不会太烦人？""会不会显得我在邀功？"另一方面，我作为小组负责人时，也常常需要逐个问进度才能知道项目实际到哪一步了。这种"双向信息盲区"让我意识到，需要一款产品来降低主动同步的心理门槛，同时让上级能一屏掌握全局。

- **大概是什么产品：** 一个支持员工端提交进度、上级端查看团队看板的Web应用。员工用进度条+阶段模板一键提交，上级通过可视化仪表盘实时查看团队整体进度，并智能标记需要关注的风险项。用温暖安全的设计语言传递"同步进度是被鼓励的"理念。

## 2. 目标用户及痛点

**面向哪些用户：**
- 核心用户：刚入职场1-3年的职场新人、性格内向/不善于向上沟通的团队成员（i人为主）。
- 次要用户：需要管理5-15人团队的项目负责人、团队领导。

**在什么场景下使用：**
- 员工完成某个阶段性里程碑后，打开「进度同步器」，通过滑条调整百分比、选择当前阶段、点选快捷话术模板，一键提交进度更新。
- 上级在晨会前或需要了解团队状态时，切换到「上级视角」看板，一屏查看所有成员的进度、同步频率、风险预警，发现滞后可一键"鼓励"而非"质问"。

**当前痛点：**
- 没有这个产品时，员工需要反复斟酌措辞、打开关闭聊天窗口，最后可能放弃沟通；或者写了消息却担心对方没有回复，不清楚进度是否真的被看到。
- 上级则需要通过发消息、开会、逐个询问来了解进展，不仅效率低下，还容易给下属造成"被催促"的压力。最终双方陷入"员工不敢问、领导不知道"的恶性循环。

## 3. 价值与意义

**效率提升：** 员工一键更新进度（从斟酌措辞到提交只需30秒），上级一屏掌握团队全局（从逐个询问到5秒浏览），预计减少团队80%的无意义进度同步沟通。每个任务自带更新记录时间线，支持追溯和删除，历史变更一目了然。

**社会价值：** 这不仅是一个工具，更是一个帮助内向型职场人建立"主动沟通"习惯的入口。通过首屏的安抚文案（"同步进度，是一件很勇敢的事"）、正向反馈机制（"鼓励一下"按钮代替质问）和可视化降低表达门槛，推动团队从"被动汇报"转向"主动协作"，建立更健康的职场沟通文化。

## 4. 创意产物 HTML 文件

本创意已使用 TRAE Work 生成可体验的创意产物 HTML 文件，文件名为 **index.html**。

该文件为完整的单页 Web 应用，包含：
- 员工端：4个进行中的任务卡片、进度条可视化、阶段标签、更新进度弹窗（含百分比滑条、阶段选择、快捷话术模板）
- 上级端：团队平均进度看板、柱状图对比、按成员/任务/风险筛选、"鼓励一下"正向反馈按钮
- 更新记录时间线：每个任务支持展开查看完整历史记录，员工可逐条删除

支持顶部导航栏一键切换「我是员工」和「我是上级」两个视角，所有交互均可完整体验。请直接上传该HTML文件到报名帖中。
